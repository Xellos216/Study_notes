# SQL Day 1 - Q&A 정리

## 1. SQL은 정보 보관하는 통인가요?
SQL은 언어이고, 정보를 보관하는 통은 데이터베이스(DB)입니다. SQL은 그 DB를 다루기 위한 도구입니다.

## 2. 로컬 DB를 다른 곳에 넘겨줄 수 있나요?
가능합니다. `mysqldump` 같은 명령어로 내보내서 다른 서버에서 복원할 수 있습니다.

## 3. 외부 공격으로 SQL이 털리면 DB 전체가 털리나요?
그럴 수 있습니다. SQL Injection 같은 공격은 잘못된 입력으로 DB 쿼리를 조작하여 민감한 정보에 접근하게 만듭니다.

## 4. Query는 무엇인가요?
Query는 DB에게 "요청하는 명령"을 의미합니다. SELECT, INSERT, UPDATE 같은 SQL 문장이 모두 Query입니다.

## 5. SQL 명령어는 왜 대문자가 많은가요?
가독성과 관례 때문입니다. SQL은 대소문자 구분을 안 하지만, 문법은 대문자, 테이블/컬럼은 소문자로 구분하는 스타일이 많습니다.

## 6. 데이터베이스와 테이블의 차이?
데이터베이스는 여러 테이블을 담는 상위 개념입니다. 비유하면 데이터베이스는 폴더, 테이블은 파일, 각 행은 엑셀 줄이라 보면 됩니다.
